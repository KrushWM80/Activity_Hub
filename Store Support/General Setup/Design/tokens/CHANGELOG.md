### BREAKING CHANGES

* remove the "component", "semantic" and "primitive"
prefixes when importing variables from js, react-native, and scss
artifacts.
* All primitive tokens are now prefixed with
"primitive" and all semantic tokens are now prefixed with "semantic".
* All primitive tokens should now be accessed with "core"
in the name.
* This removes the semantic token for field components'
border radius. Use the primitive border radius tokens.
* This updates the previous "yellow" accent hue from
`color.core.spark` to `color.core.yellow`.
* primary and destructive actions no longer have border
tokens. Secondary and tertiary actions have localized border tokens.
* This removes `color.loading.inverse` and adds
`color.loading.white`. Related: https://jira.walmart.com/browse/LD-5330
* **DataTable:** use `aliasName` and `aliasOptions` instead of the font
token names.
* **DataTable:** This updates the interactive token names for Data
Table. Use focused, hovered, pressed, and activated.
* **DataTable:** Data Table component tokens are no longer themeable.
Override the semantic tokens to theme the component.
* **DataTable:** deprecated Data Table tokens are no longer supported.
Migrate to their updated token names/components.
* **DatePicker:** Remove "default" on Date Picker interactive state
token names.
* **Tag:** a majority of token names have changed to make Tag
adopt semantic color tokens. Consult documentation/pull requests to
make updates.
* **TextField:** This replaces enabled state tokens with "_" per
authoring convention. Replace "-default" with "" to migrate.
* **Link:** remove "default" from the token name
* **Checkbox:** This updates interactive state names to align with the
data dictionary.

* access "enabled" tokens by removing "default" from their name
* Update "checked" to "activated"
* **Button:** access the "enabled" (formerly "default") token by
omitting "default" from the name.
* **Button:** reference fontWeight on the Button's textLabel instead
of variant.
* **Button:** reference borderWidth on the Button's container instead
of variant.
* **Button:** reference Button's textLabel for text-related tokens.
* **Alert:** component.alert.actionButton.textDecoration.default has
been changed to component.alert.actionButton.textDecoration._.
* **Chip:** This fixes Chip's height for the large and small size.
It also moves height from `line-height` on the textLabel to `height` on
the container.
* **Chip:** remove usage of activated backgroundColor and
textColor.
* **Chip:** This renames interactive state names to align with the
data dictionary.

* `default`: remove from token name
* `focus`: update to `focused`
* `selected`: update to `activated`
* color.navigation.text.onFill.activated is now
color.navigation.text.onFill.activated._
* This removes support for `zIndex` tokens based on a team
decision that `zIndex` only applies to the web platform. To migrate: inline
a collection of possible `z-index` values within your product.
* color.core.transparent has been renamed to
color.core.transparentDark.0.
* Accent semantic tokens now require "accent" in their path.
* `color.separator._` is now `color.separator`.
* This aligns the scrim token name to Figma. To update:

* Rename `color.surface.scrim` to `color.scrim`.
* Accessing some tokens now requires a "_" suffix.
* this updates the old globals to the new primitive
design tokens. The new primitive design tokens contain most values from
globals, but the names have changed.
