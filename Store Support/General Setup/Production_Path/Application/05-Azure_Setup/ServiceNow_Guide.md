# ServiceNow: Azure Resource Requests

## 🔗 ServiceNow for Azure Setup

Azure subscription and resource requests are submitted through ServiceNow:

- **Main Portal**: https://walmartglobal.service-now.com/
- **Service Catalog**: Access cloud services requests
- **Access**: Requires Walmart authentication

---

## 📋 Using ServiceNow for Azure Requests

### Azure Subscription Request:

1. **Access ServiceNow**
   - [ ] Go to https://walmartglobal.service-now.com/
   - [ ] Log in with Walmart credentials
   - [ ] Navigate to Service Catalog

2. **Find Cloud Services**
   - [ ] Search for "Azure subscription" or "Cloud services"
   - [ ] Select appropriate catalog item
   - [ ] Review requirements and prerequisites

3. **Submit Request**
   - [ ] Complete all required fields
   - [ ] Include business justification
   - [ ] Reference your **APM ID** and **SSP approval**
   - [ ] Specify environment (Dev/Test/Prod)
   - [ ] Submit request and note RITM#

4. **Track Request**
   - [ ] Monitor request status in "My Requests"
   - [ ] Respond to any questions from cloud team
   - [ ] Await subscription provisioning

## 📊 ServiceNow Azure Request Workflow

### Common Azure Requests via ServiceNow:

**Subscription Requests:**
- New Azure subscription
- Subscription access modification
- Cost center assignment
- Management group placement

**Resource Requests:**
- Virtual machine provisioning
- Storage account setup
- Database service requests
- Networking configuration
- Load balancer setup

**Support Requests:**
- Technical issues
- Permission problems
- Cost management questions
- Resource limit increases

## 📝 ServiceNow Azure Best Practices

### Before Requesting:
- ✅ Have **SSP approval** from Step 4
- ✅ Know your cost center and budget
- ✅ Identify required Azure services
- ✅ Have architecture diagram ready
- ✅ Understand naming conventions

### During Request:
- ✅ Provide complete information upfront
- ✅ Include business justification
- ✅ Reference APM ID and SSP
- ✅ Specify correct environment tags
- ✅ Document resource requirements clearly

### After Provisioning:
- ✅ Verify subscription access
- ✅ Document subscription ID
- ✅ Set up cost alerts
- ✅ Apply required resource tags
- ✅ Configure monitoring

## 📞 ServiceNow Azure Support

## 🔗 Key Information to Track

### From ServiceNow Requests:
- **RITM#** (Request Item number)
- **Submission date**
- **Approval status**
- **Provisioning date**
- **Azure Subscription ID** (once assigned)
- **Contact person** from cloud team

---

## ⚠️ Important Notes

- **SSP approval required** before production Azure requests
- **Budget approval needed** for Azure resources
- **Naming conventions** must follow Walmart standards
- **Resource tags** are mandatory (CostCenter, Environment, Owner, etc.)
- **Security groups** required for access management

All Azure provisioning and support happens through ServiceNow Service Catalog!
