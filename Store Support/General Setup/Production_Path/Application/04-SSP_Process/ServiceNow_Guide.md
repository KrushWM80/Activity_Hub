# ServiceNow: SSP Portal

## 🔗 SSP Portal Access

The Solution Security Plan (SSP) is initiated and managed entirely through ServiceNow:

- **SSP Portal**: https://walmartglobal.service-now.com/ssp?id=ssp_plan_form
- **Access**: Requires Walmart authentication and APM "Certified" status

---

## 📋 Using ServiceNow SSP Portal

### SSP Initiation:

1. **Access SSP Portal**
   - [ ] Go to https://walmartglobal.service-now.com/ssp?id=ssp_plan_form
   - [ ] Log in with Walmart credentials
   - [ ] Verify you have your **APM ID** from Step 3

2. **Start New SSP**
   - [ ] Click "Start New SSP" or similar action
   - [ ] Enter your **APM ID** to link SSP to your APM record
   - [ ] This auto-populates some information from APM

3. **Complete Solution Details**
   - [ ] Fill in solution overview section
   - [ ] Add architecture and data flow details
   - [ ] Document security controls
   - [ ] Provide integration points
   - [ ] Upload supporting documentation (diagrams, etc.)

4. **Submit for Review**
   - [ ] Review all entered information
   - [ ] Submit SSP
   - [ ] Note your SSP reference number
   - [ ] System will auto-generate assessment tasks

---

## 📊 ServiceNow SSP Workflow

### Auto-Generated Tasks:

After SSP submission, ServiceNow automatically creates tasks based on your APM data classification:

**Tasks You'll See:**
- **Privacy Assessment** - Always required
- **SOX Controls** - If SOX data classification in APM
- **Data Governance Review** - Always required
- **Vendor Assessment** - If third-party vendors involved
- **TCA (Technical Compliance Assessment)** - Always required
- **Other** - Based on your specific compliance requirements

### Monitoring Tasks in ServiceNow:

1. **Access Your SSP**
   - [ ] Return to SSP Portal
   - [ ] View your active SSP
   - [ ] See all auto-generated tasks

2. **Check Task Status**
   - [ ] Each task shows: Not Started / In Progress / Complete / Approved
   - [ ] Assigned SME team name
   - [ ] Questions from analysts (if any)
   - [ ] Required actions

3. **Respond to Questions**
   - [ ] Monitor SSP daily for new questions
   - [ ] SMEs/TCA analysts will post questions in task comments
   - [ ] Respond within 2 business days
   - [ ] Provide complete answers with supporting documentation

4. **Track to Approval**
   - [ ] Watch for all tasks to reach "Approved" status
   - [ ] System evaluates when all tasks complete
   - [ ] Receive "SSP Approved" notification
   - [ ] Production deployment authorized

## 📝 ServiceNow SSP Best Practices

### Navigation Tips:
- **Bookmark SSP Portal** for daily checks
- **Set up email notifications** for SSP updates
- **Check SSP at least once per day** during active review
- **Use task comments** to communicate with SMEs

### Communication in ServiceNow:
- **Use task comments** for questions and clarifications
- **Attach documents** directly to tasks when requested
- **Tag analysts** if urgent clarification needed
- **Keep responses professional and complete**

### Common ServiceNow SSP Actions:
- **View SSP status**: Check main SSP record
- **Access individual tasks**: Click on each auto-generated task
- **Upload documents**: Use attachment feature in SSP or tasks
- **Check history**: View audit trail of all changes and responses

### Troubleshooting:
- **Tasks not auto-generated**: Verify APM ID is correct and APM is "Certified"
- **Can't access SSP Portal**: Verify you're Application Owner or Technical Owner
- **Task stuck in review**: Use task comments to follow up with SME team
- **Questions unclear**: Ask for clarification in task comments immediately
- **Need escalation**: Contact Secrisk@wal-mart.com

## ⚠️ Critical ServiceNow SSP Notes

### Before Initiating SSP:
- ✅ Must have **APM "Certified" status** from Step 3
- ✅ Have **APM ID** ready to enter
- ✅ Prepare architecture diagrams and documentation **before starting**
- ✅ Plan for 1-3 weeks of active monitoring and responses

### During SSP Review:
- ✅ **Check ServiceNow daily** for new questions
- ✅ **Respond within 2 business days** to all questions
- ✅ Provide **complete answers** with supporting documentation
- ✅ Don't let tasks sit idle - delays compound across multiple reviews

### Response Quality Matters:
- ✅ Complete and thorough answers accelerate approval
- ✅ Partial answers lead to follow-up questions (more delays)
- ✅ Supporting documentation helps analysts understand your solution
- ✅ Architecture diagrams are especially valuable

## 📈 ServiceNow SSP Timeline Expectations

| Phase | Actions in ServiceNow | Expected Duration |
|-------|----------------------|-------------------|
| **SSP Initiation** | Submit SSP form with APM ID | Same day |
| **Task Generation** | System creates assessment tasks | 1-2 days |
| **SME Review** | Analysts review and post questions | 5-10 business days |
| **Your Responses** | Answer questions in task comments | 2 business days per round |
| **Final Approval** | All tasks marked "Approved" | 2-3 days after tasks complete |
| **Total** | Full SSP process | **1-3 weeks** |

**💡 Pro Tip:** Faster, more complete responses in ServiceNow = faster SSP approval!
