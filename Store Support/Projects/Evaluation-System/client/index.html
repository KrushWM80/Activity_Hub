Hosting Platform Overview

Understand your options for deploying the Evaluation System and Activity-Hub platform.

Code Puppy Pages

❌ Limited Use
✓ Quick Setup - No approval needed
✗ Static Only - No live data
✗ No BigQuery - Can't access real-time data
✓ No Cost - Free platform

Best For: Static documentation, archived reports, reference guides only. NOT suitable for active platforms requiring live data.

⚠️ Note: Cannot support the Evaluation System's real-time evaluation features or Activity-Hub's live data requirements.

Posit (RStudio Connect)

⚠️ Conditional
✓ Analytics Ready - Built for R/Python
⚠ User Limits - Restricted concurrent users
⚠ SSP Required - Security plan needed
⚠ Approval Timeline - Weeks for approval

Requirements: Updated Security & Safeguards Plan (SSP), compliance documentation, user limit agreement

Best For: R/Python analytics, smaller teams (<1,000 users), interactive dashboards

Self-Hosted Server

✓ Full Control
✓ Complete Independence - Full control of platform
✓ BigQuery Support - Full data access
✗ You Own Infrastructure - Server & resources your responsibility
✗ Ops Overhead - Monitoring, maintenance, security

Requirements: Physical/Virtual server, IT ops resources, network security compliance, monitoring capabilities

Best For: Organizations with existing on-premise infrastructure, custom security needs, isolated deployments

Walmart Internal Resources

✅ Recommended
✓ BigQuery Included - Full data warehouse access
✓ Enterprise Scale - Supports 50,000+ users
✓ Professional Support - Walmart IT teams help
⚠ Approval Required - Platform governance review

Requirements: Platform approval, current SSP, architecture review, executive sponsorship, compliance certification

Best For: Enterprise-scale platforms (50,000+ users), long-term solutions, systems requiring BigQuery integration, Walmart-critical systems

✅ RECOMMENDED FOR ACTIVITY-HUB - Only realistic option for full enterprise scale and live data integration

MyWM Experiments

✓ Store-Level Reach
✓ Immediate Access - All associates at store level and above
✓ Agent Support - Agent-driven platform capabilities
✓ Built-In Audience - 50,000+ ready users
✓ Rapid Rollout - Quick deployment to stores

Best For: Store-facing initiatives, rapid experimentation and testing, large-scale user adoption (all stores), paired with backend platform

✅ RECOMMENDED FRONTEND for Activity-Hub user interface and distribution

Me@Campus

⚠️ Limited Audience
✓ Market-Level Focus - Designed for district managers and above
✗ No Store-Level Access - Cannot reach individual store managers
✓ Management Dashboards - Good for executive views

Best For: Market-level decision dashboards, market manager tools, executive summaries only

⚠️ Note: Limited audience scope - doesn't support store-level initiatives.

Power Apps (Microsoft Power Platform)

✓ Active Development
✓ 3 Active Apps - Currently deployed solutions
✓ Proven ROI - Reducing real friction
⚠ Market-Level Only - Current audience limitation
✓ Expansion Possible - Can expand to store managers

Current Deployment:

  • Quarterly Operations Checklist (Store Mgr ↔ Market Mgr)
  • Feedback Ticket Submission (Store Mgr → Home Office)
  • Special Feature Order Management (Seasonal/Event items)

Best For: Multi-functional business apps, structured workflows, Microsoft ecosystem integration (Excel, Teams, Dynamics)

🎯 HIGH-VALUE OPPORTUNITY: Expand to store managers to eliminate friction and increase adoption (lower complexity than full platform approvals)

How to Choose Your Hosting Option

For the Activity-Hub Platform (50,000+ users)

Recommended Architecture:

  1. Backend: Walmart Internal Resources (WM Cloud)
    • Enables enterprise scale and BigQuery integration
    • Requires platform approval + SSP (4-12 weeks)
    • Only realistic option for 50,000+ users with live data
  2. Frontend/Distribution: MyWM Experiments
    • Immediate access to store-level users
    • Built-in platform for rapid rollout
    • Integrated with Walmart app ecosystem
  3. Special Workflows: Power Apps (expansion)
    • Structured business processes
    • Store manager tools and feedback collection
    • High-value opportunity to eliminate friction

For Smaller Initiatives (<1,000 users)

Consider:

  • Posit - If analytics-focused, smaller team, can wait 2-3 weeks
  • Power Apps - If workflow/business process focused, needs approval
  • Self-Hosted - If you need full control and have infrastructure resources

Before Choosing: Key Questions

  1. How many users? Draft: <100 (any), Small: <1,000 (Posit/Power Apps), Enterprise: 50,000+ (WM only)
  2. Need live data? Yes → Requires WM Internal, MyWM, or Self-Hosted. No → Code Puppy Pages okay
  3. Timeline? Urgent (weeks) → Self-Hosted/Code Puppy. Patient (months) → WM Internal (best option)
  4. Infrastructure? Don't have → WM Internal. Have → Self-Hosted option
  5. Walmart-critical? Yes → WM Internal (required for governance). No → Other options viable

Next Steps

  1. Answer the 5 key questions above
  2. Review the recommended platform for your situation
  3. Check the "Requirements" section for that platform
  4. Contact platform governance/IT teams with your plan
  5. Begin approval process (4-12 weeks for enterprise platforms)
