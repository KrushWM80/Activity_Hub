Understanding Fix Cards

Each pending fix displays detailed information about the reported issue:

Field Description
Fix ID Unique identifier (e.g., FIX-001)
Category Type of issue (filter_issue, display_bug, etc.)
Priority HIGH, MEDIUM, or LOW
Description Summary of the reported issue
Root Cause AI's analysis of what caused the problem
Investigation Notes Step-by-step analysis the AI performed
Files Analyzed List of code files the AI examined
Current Code Existing code that will be replaced
New Code Proposed replacement code
⚠️ NEEDS REVIEW Badge: When you see this badge, the AI flagged the issue as complex and requires extra careful review before approval.

Taking Action

✅ Approve

When to use: The analysis correctly identifies the root cause and the proposed fix is appropriate.

Result: Code change is automatically applied to the specified file.

❌ Deny

When to use: The analysis is incorrect, fix wouldn't solve the problem, or change could cause other issues.

Result: Fix is marked as denied and moved to history. No code changes are made.

⏸️ Hold

When to use: Need more time to review, want to consult with someone, or requires additional investigation.

Result: Fix remains in the pending queue with "On Hold" status.

Review Checklist

Before approving any fix, verify the following:

  • Read the Investigation Notes - understand how AI reached its conclusion
  • Compare Current Code vs New Code - ensure change is appropriate
  • Check the File Path - confirm the correct file is being modified
  • Look for ⚠️ NEEDS REVIEW badge - extra caution required
  • Consider side effects - could this change affect other features?

When to Escalate

Contact the development team if:

  • Changes involve authentication or security code
  • Multiple files need modification
  • Code change is more than 15 lines
  • You're unsure about the potential impact
  • Fix marked with ⚠️ NEEDS REVIEW and you're not confident

Troubleshooting

Problem Solution
"Fix application failed" The code may have changed since analysis. Contact dev team to manually apply.
"File not found" Target file may have been moved or renamed. Deny and request new analysis.
Dashboard not loading Verify server is running at localhost:8001/api/health
Can't login Clear browser cookies and try backup credentials
