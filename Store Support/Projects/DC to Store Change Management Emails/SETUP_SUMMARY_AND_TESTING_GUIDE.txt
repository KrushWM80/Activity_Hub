3. HOW EMAILS ARE DISTRIBUTED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP-BY-STEP PROCESS:

Step 1: Detect Changes (Hourly)
  └─ Compare today's SDL export vs yesterday's
  └─ Identify: Store Managers, Market Managers, Region Managers

Step 2: Map to DCs
  └─ Use dc_to_stores_lookup.json
  └─ Example: Store 1234 → DC 6020 (Ambient) + DC 6040 (Perishable)

Step 3: Group by DC
  └─ Collect all changes affecting each DC territory
  └─ Create one email per DC with their specific changes

Step 4: Generate Email
  └─ Create beautiful HTML with manager changes
  └─ Show old → new manager with styling
  └─ Include SDL links for store lookups

Step 5: Determine Recipients
  └─ FOR EACH DC: Get their GM and AGM emails
  └─ Example DC 6020: [6020GM@email.wal-mart.com, 6020AGM@email.wal-mart.com]

Step 6: Send Email
  ✓ In TEST MODE: 
    └─ SENDS TO: kendall.rush@walmart.com
    └─ SHOWS: Production recipients list (for visibility)
  
  ✓ In PRODUCTION MODE (when TEST_MODE=False):
    └─ SENDS TO: Each DC's GM and AGM (via BCC)
    └─ Example recipients per email:
        • 6020GM@email.wal-mart.com
        • 6020AGM@email.wal-mart.com

4. EMAIL RECIPIENT PATTERNS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

👥 RECIPIENT FORMAT: <DC_NUMBER><ROLE>@email.wal-mart.com

Examples:
  • 6020GM@email.wal-mart.com   = DC 6020 General Manager
  • 6020AGM@email.wal-mart.com  = DC 6020 Assistant General Manager
  • 6040GM@email.wal-mart.com   = DC 6040 General Manager
  • 6040AGM@email.wal-mart.com  = DC 6040 Assistant General Manager
  • 6080GM@email.wal-mart.com   = DC 6080 General Manager
  • 6080AGM@email.wal-mart.com  = DC 6080 Assistant General Manager

📍 DC Numbers in System: 6020, 6040, 6080, 6090, 6120, 6140, etc.

🔐 EMAIL DELIVERY METHOD: BCC (Blind Carbon Copy)
  └─ Recipients can't see each other's addresses
  └─ Prevents accidental reply-all floods

7. SCHEDULE CHANGE: HOURLY → BI-WEEKLY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CURRENT SCHEDULE: Hourly (for testing)
  └─ Task Scheduler runs daily_check_smart.py every hour
  └─ Idempotent: Only processes once per calendar day
  └─ Frequency: ~24 runs/day (but email sent max 1x/day)

FUTURE SCHEDULE: Bi-Weekly (every 2 weeks on pay cycle)
  └─ When switching: Update Windows Task Scheduler
  └─ Option 1: Specific dates (every 2nd Friday)
  └─ Option 2: Custom schedule aligned with HR pay cycles
  └─ Reduces email noise while maintaining visibility
  └─ Better for leadership review cycles

🔧 TO IMPLEMENT BI-WEEKLY SCHEDULE:
  1. Identify exact pay cycle dates
  2. Update: setup_hourly_task_auto.bat
  3. Change: /SC HOURLY → /SC WEEKLY + /D FRI + manual date logic
  4. Test: Run on first pay cycle date
  5. Monitor: Confirm emails sent on correct schedule
